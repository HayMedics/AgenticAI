Analyzing Retail Sales with SQL  
I've built a Retail Store Management System using SQL, tracking customers, stores, and products. The project includes: - Database Design: Created tables for customers, stores, products, and orders - Data Analysis: Answered 10 project questions, including total revenue by store, products sold by category, and top-spending customers  
- SQL Techniques: Used INNER JOIN, LEFT JOIN, RIGHT JOIN, GROUP BY, HAVING, and subqueries to extract insights  
Some of the key queries I wrote include: - List all customers with their city and email: `SELECT customer_name, city, email  
FROM customers;`  
Show all orders with the customer name, product name, and store name: `SELECT  
c.customer_name, p.product_name, s.store_name  
FROM orders o JOIN customers c ON  
o.customer_id = c.customer_id JOIN products p  
ON o.product_id = p.product_id JOIN stores s  
ON o.store_id = s.store_id;`  
- Find total revenue generated by each store:  
`SELECT s.store_name, SUM(p.price *  
o.quantity) AS total_revenue FROM orders o  
JOIN products p ON o.product_id =  
p.product_id JOIN stores s ON o.store_id =  
s.store_id GROUP BY s.store_name;

StrokeGuard AI — Clinical Stroke Risk Prediction Web App  
StrokeGuard AI is an end-to-end machine learning web app that estimates a patient's stroke risk from routine clinical inputs (age, blood pressure history, glucose, BMI, smoking status and more). I designed, built and deployed it from scratch to demonstrate responsible, explainable ML in healthcare.   
Highlights:  
-	Built a leak-free scikit-learn pipeline (imputation, scaling, encoding) with a calibrated logistic regression model - Handled severe class imbalance (only ~5% of patients had a stroke) using class weighting - Chose a cost-based decision threshold that weights a missed stroke 10x a false alarm, prioritising sensitivity as a screening tool should  
-	Added explainability (SHAP-style feature contributions), Monte-Carlo confidence intervals, a what-if simulator, population comparison, batch CSV screening, session history, a model-validation page and branded  
PDF reports  
-	Documented in the spirit of the TRIPODAI reporting standard and deployed live on  
Streamlit Cloud  
Honest performance on a held-out test set: ROC-AUC approximately 0.84 with high negative predictive value. It is an educational screening tool, not a medical device.    
Coming from a medical background, this project sits exactly where I want to work: the bridge between clinical knowledge and machine learning.
